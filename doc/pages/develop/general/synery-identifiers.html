Synery Identifiers

Interface Booster uses "Synery Identifiers" for a lot things.

For example for:

  • Endpoint names
  • Field names
  • Question/Parameter names
  • Filter names
  • GET-Value names
  • All Paths

Pattern

All Synery Identifiers must:

  • Start with a lower or uppercase letter or an underscore "_"
  • Only contain lower or uppercase letters, underscores "_" or numbers
