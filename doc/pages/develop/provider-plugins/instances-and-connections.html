Instances and Connections

A Provider Plugins can have multiple instances. Those implement the interface IProviderPluginInstance.

This allows the developer of a Provider Plugin to create multiple versions of the plugin for example to work with different API versions of a foreign system or different versions of a file format.

The instance of a Provider Plugin can "ask" the interface developer a few questions needed to open a connection. Answering a questions can be optional or required.

Step-by-Step: How is a Connection created?

The Interface Booster performs the following steps to open a connection:

Read plugin.xml from the plugins' main directory.
Load the *.DLL assembly that matches the selected version.
Search for the class that implements IProviderPlugin.
Call CreateProviderPluginInstance to get a IProviderPluginInstance using the GUID from the plugin.xml file.
Get the ConnectionSettingQuestions.
Check the parameter (answers) set by the interface developer: Are all required parameters set? Do the data types match?
Call CreateProviderConnection to get a IProviderConnection.
The Connection whit its' Endpoints is now available.
