Requests and Responses

Interface Booster sends requests to Provider Plugins endpoints and expects them to return a response.

Step-by-Step: How a request works?

Before Interface Booster can perform a request a connection must be available. See: Instances and Connections

Assume we already have an open connection the following steps are part of a request:

A data command with an indefinite number of nested commands is given.
Search for a connection that matches the given endpoint path.
Prepare the request data:
Requested fields on read
Filters on read or delete
Export schema on write
Parameters (always)
Requested values on execute
Validate the request data: Are all required parameters and fields set? Do the data types match?
Call the Run***Request method on the Endpoint.
Wait until the Provider Plugin returns a response.
Read back the returned data to memory (variables) or database (record sets).
End of request.
