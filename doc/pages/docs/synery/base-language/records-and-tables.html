Records and Tables draft

A record is a composition of values. It is an instance of a record type. The record type specifies what fields a record can have. On the other side are tables. A table consists of rows and columns (fields).

Records are stored in memory (RAM) and tables in the Synery Database, which stores the data on the harddisk

Primitive values of a record can be stored in a table row and the way arround the values of a table row can be loaded into a record.

For the mapping between a row and a record the field names are used. If a field only exists in one of both the field value is left NULL.

ADD

With ADD you can append a new record (see: Record Types) to a table.

If the table doesn't already exists it is created based on the definition of the Record Type.
