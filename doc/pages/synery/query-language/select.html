Select draft

Like in SQL the SELECT command is used to chose fields and restructure the schema of the result set.

One query statement can contain multiple SELECT commands. Each command is based on the result set of the prior commands. For example if the first SELECT command doesn't select a field from the original table this field cannot be used anymore in the following SELECT commands. You can think like you have a new table after every command in a query statement.

An Example

Assume we have a table like this:

\people
Id (INT) IsMale (BOOL) Firstname (STRING) Lastname (STRING) Size (INT)
15 TRUE Mike Meyer 182
22 FALSE Sandra Smith 168
27 TRUE Peter Muller 169
38 FALSE Rachel Lopez 175

Now let's have a look how a Synery query could look like when querying the table from above:

On the first line you see a table statement with an equality sign =. This mean that the result of the following query should be stored in the table with the given name. In this case a new table with the name \myTables\myPeople will be created.

On the second line we have a FROM command that declares which is the source table (in this case \people) an what alias (here: p) should be used for that table.

On the third line we have a SELECT command that defines the structure of the resulting table.

The result will look like this:

\myTables\myPeople
FullName (STRING) Size (INT)
Mike Meyer 182
Sandra Smith 168
Peter Muller 169
Rachel 175
