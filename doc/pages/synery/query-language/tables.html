Tables draft

The Synery query language works with tabular data structures. This means that you have columns that represent fields and rows that represent records.

Tables can be addressed by using a so called "Table Statement" (see below).

Technically it is important to know that there is a big different between variables and tables because they are stored in different ways. Variables always stay in the computers memory during runtime. On the other hand tables are stored on the hard disk and are only partially loaded to the memory during a query operation.

Table Statement

In Synery you can address a table by using a table statement. A table statement looks like a path in a network environment. An example:

It is different to variable identifiers because it always starts with a backslash \. The benefit of this is that you always can distinguish between variables and tables.

You are free to use as much backslashes \ as you want in a table statement. It is recommended that you use these paths to organize and group your tables. A few examples:

  • \myERP\resources\employees
  • \myERP\resources\employees\addresses
  • \myERP\resources\employees\contracts
  • \myERP\resources\facilities
  • \myERP\finance\accounts
  • \myERP\finance\bookings
  • \XmlFile\people
  • \XmlFile\addresses
