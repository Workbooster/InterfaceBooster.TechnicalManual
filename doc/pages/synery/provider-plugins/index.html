Provider Plugins draft

Provider Plugins allow Synery developers to easily exchange data with a foreign system or a file by using some commands. These plugins offer you a standardized way to import and export data from and to the Interface Booster environment.

The usual way you work with data in Interface Booster:

  1. Connect to some foreign systems and/or files.
  2. Import some data (usually into some tables).
  3. Modify the data (filter, group, join etc.).
  4. Export the data.

The Provider Plugins help you to create a connection to the external data source, provide some endpoints to load data from and offer some endpoints to send data to.

Terms

The following terms are used in connection with Provider Plugins.

Provider Plugin
A Provider Plugin is a DLL assembly that extends the Interface Booster. It may provide access to a foreign system like a database, an ERP-System, a Web Service or either a specific file format like XML, CSV etc.
Connection
Before you can start to exchange data with an external data source you have to open a connection. For example if you'd like to work with a database you may have to specify a server, a user-name and a password. Or for example if you'd like to read or write a file you may have to set a file-path.
Command
There are a lot of commands in Synery. But in this context usually the import/export commands are meant.
Synery supports CRUD(E) commands:
  • CREATE
  • READ
  • UPDATE
  • DELETE
  • EXECUTE
Endpoint
In every import/export command (READ, CREATE etc.) you have to specify an endpoint you would like to exchange data with. Each endpoint has one primary resource that specifies what arguments can be used in the command.
Resource
A resource specifies the arguments that can be used in a command. Possible arguments are:
  • a schema
  • some parameters
  • some filter-parameters
  • some fields
  • some get-values
  • some nested resources
Request
If a command is executed and some arguments are sent to an endpoint provided by a provider plugin connection this is called a "request".
