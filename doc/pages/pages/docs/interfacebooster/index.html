Interface Booster Runtime

The Interface Booster Runtime environment is a software that is used to execute an interface application.

System Context

The following System Context UML diagram shows how the components of the Interface Booster environment are related to each other:

An interface job may be started by an End-User or by a Batch Job. On startup the runtime software needs to know the filepath of an Interface Definition and the name(s) of the job(s) that should be executed. The Interface Definition contains the business logic. It is implemented by using the Synery programming language. An Interface Definition may contain many Synery code files.

The Interface Booster Runtime software itself doesn't know any foreign system. To communicate with an other system Interface Booster uses Provider Plugins.

For extending the functionality of the Synery language there are Library Plugins.

What is needed for starting an interface job?

For executing an interface job there are a few components required:

  • Interface Booster Runtime Software (EXE)
  • The Interface Definition:
    • importDefinition.xml
    • Synery Code files (*.sny)
  • All referenced Provider Plugins
  • All referenced Library Plugins
