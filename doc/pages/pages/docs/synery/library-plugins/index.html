Library Plugins draft

Library Plugins allow Synery developers to easily extend the base functionality of Synery. With the knowledge of a .NET programming language like C# or VB.NET you can implement additional functions, that can be used within Synery but are executed as .NET Common Language Runtime Code. Therefore everything you can do in .NET you can also do within Synery.

The only thing a Synery developer needs to do, is to reference the Library Plugin in the Interface Definition.

Terms

The following terms are used in connection with Library Plugins.

Library Plugin
A Library Plugin is a DLL assembly that extends the Interface Booster. It may provide some static functions and some static variables that can be used in Synery code.
Static Function
It is a function that can be used in Synery code. A function can take primitive data types as parameters and can also return a primitive typed value.
From the .NET-view a static function is a public method (marked by a [StaticFunction]-attribute) of a class that implements the IStaticExtension interface and has no constructor.
Static Variable
It is a variable that can be used in Synery code to get or set a value. A variable can be a primitive data type. It can be read-only or write-only.
There are many use cases for static variables. For example a read-only variable could represent the current date. Or a variable could be used to specify the recipients of an E-Mail in an e-mail Library Plugin. From the .NET-view a static variable is a public property (marked by a [StaticVariable]-attribute) of a class that implements the IStaticExtension interface and has no constructor.
