Endless possibilities thanks to plugins

The Interface Booster environment is based on a strong plugin ecosystem. The idea is simple: Write a plugin once - use it everywhere.

Usually you are using at least two plugins: One for the data source and one for the data receiver. But there are even more reasons to build your interface based on Interface Booster Plugins:

  • Save time and money.
  • Re-use existing logic and functionality in other projects.
  • No need to re-invent the wheel.
  • If the environment changes there is only one place where the change must be applied.
  • Maybe someone else has already written a plugin that matches your needs.
  • Everything you can do with the Microsoft .NET Framework you can do in Interface Booster by using a plugin.

Provider Plugins

Provider Plugins are used to import and export data from and to foreign systems or files. In the following section you find a list of available Provider Plugins.

If the Provider Plugin you are looking for isn't available yet you can implement your own plugin. See: Develop > Provider Plugins. Once created, please let us know, so we add it to our list and can help you to enhance the awareness of your product.

Library Plugins

With Library Plugins it is possible to easily extend the base functionality of Synery, the script language used in Interface Booster. Usually this are some helpers like conversion functions (e.g. date formatting function) or static variables (e.g. current day of week)

With the knowledge of a .NET programming language like C# or VB.NET it is possible to implement additional functions, that can be used within Synery but are executed as .NET Common Language Runtime Code. If the Library Plugin you are looking for isn't available yet you can implement your own plugin. See: Develop > Library Plugins.
